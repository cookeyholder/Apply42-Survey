# 欄位名稱差異報告（ODS vs 程式碼）

比對範圍：
- ODS：高雄高工四技二專甄選入學志願調查系統_2026年版.ods
- 程式碼：專案根目錄所有 .gs 與 .html

比對重點：
- 參數設定（A=參數名稱、B=參數值、C=參數說明）
- 考生志願列表（第一列欄位名稱）

## 一、參數設定比對結果

程式碼使用到的 `configs[...]` 參數鍵值：
- 學校名稱
- 系統名稱
- 系統關閉時間
- 承辦單位
- 聯絡電話
- 電子郵件
- 是否寄送選填內容通知信
- 通知信寄件人名稱
- 通知信主旨
- 通知信內容標題
- 報名學校代碼

結論：
- 與 ODS「參數設定」A 欄參數名稱清單一致。
- 未發現缺漏或拼字不一致。

## 二、考生志願列表比對結果

程式碼以 `indexOf(...)` 查找的重點欄位：
- 統一入學測驗報名序號
- 班級名稱
- 考生姓名
- 報考群(類)代碼
- 報考群(類)名稱
- 繳費身分
- 報名費
- 是否參加集體報名
- 志願1校系名稱
- 志願1校系代碼

ODS「考生志願列表」第一列對應欄位（節錄）：
- 志願1代碼、志願2代碼 ... 志願6代碼
- 志願1校系名稱、志願2校系名稱 ... 志願6校系名稱

差異項：
- 程式碼使用：志願1校系代碼
- ODS 欄位為：志願1代碼

影響評估：
- `statistics.gs` 目前以「志願1校系代碼」作為統計起始欄位，若工作表實際為「志願1代碼」，可能造成欄位索引錯置或回退邏輯非預期。

建議修正：
1. 以「志願1代碼」作為代碼欄位起始索引（符合 ODS 現況）。
2. 保留 fallback：若找不到「志願1代碼」再嘗試「志願1校系代碼」，提升相容性。
3. 在修補任務中加入欄位存在檢查與明確錯誤訊息。

## 三、結論

- 參數設定：一致。
- 考生志願列表：發現 1 項高關聯差異（志願1校系代碼 vs 志願1代碼）。
- 建議在本次第一優先修補（統計正確性）一併處理。
