## Why

目前統計頁雖已具備延遲載入與快取，但切換到統計頁時仍是一次載入全量資料並集中渲染，資料量增加時容易讓老師與學生等待過久。現在需要先把統計體驗調整為效能優先，再逐步提升視覺化可讀性。

## What Changes

- 將統計載入流程改為分層：先載入摘要，再按群類載入明細。
- 新增統計結果語意化欄位，將志願顯示從純代碼提升為「校系代碼 + 校系名稱」。
- 定義圖表呈現策略（先以 Top N 長條圖 + 明細表為主），避免首屏載入過重。
- 建立統計效能預算與快取分層策略，明確規範首屏回應時間與 payload 上限。
- 保留表格為穩定 fallback，確保在低效能情境仍可快速檢視與對帳。

## Capabilities

### New Capabilities
- `statistics-progressive-delivery`: 統計資料改為摘要先行、群類明細後載的漸進式交付。
- `statistics-semantic-labeling`: 統計項目提供校系代碼與名稱的語意化顯示與一致映射。
- `statistics-performance-budget`: 建立統計載入、渲染與快取的效能預算與驗收門檻。

### Modified Capabilities
- 無。

## Impact

- 受影響系統：Google Apps Script 統計資料聚合流程、學生端與老師端統計頁面。
- 受影響檔案：統計後端資料函式、統計頁前端渲染邏輯、快取策略與設定。
- 風險面向：資料映射一致性、快取失效策略、前端渲染節點數控制。
- 外部相依：不新增外部相依套件，優先使用現有 Apps Script 與前端能力。
