# 階段 A 驗收清單（桌機 + 手機）

## A. 視覺一致性
- [ ] A1 卡片圓角、陰影、頂部強調線在三頁規則一致。
- [ ] A2 主要按鈕互動（hover/active/disabled）符合統一規則。
- [ ] A3 表單 focus ring 與錯誤樣式符合統一規則。
- [ ] A4 狀態文案語氣一致（載入中、空資料、錯誤、已截止）。
- [ ] A5 倒數區在學生頁與老師頁樣式一致。

## B. 志願驗證回饋（學生頁）
- [ ] B1 未選任何志願時，送出按鈕停用且顯示「請至少選擇一個志願」。
- [ ] B2 選到重複志願時，對應欄位出現紅框與錯誤文字。
- [ ] B3 重複志願時，表單上方摘要列出「有重複志願」提示。
- [ ] B4 同校超過上限時，超限欄位出現紅框與上限提示。
- [ ] B5 超上限時，表單上方摘要顯示對應學校上限訊息。
- [ ] B6 修正後錯誤回饋會即時清除（欄位 + 摘要）。

## C. 桌機驗收（Chrome）
- [ ] C1 Chrome 最新版測試通過。
- [ ] C2 錯誤欄位可自動捲動至可視位置。
- [ ] C3 修正錯誤後可恢復按鈕可送出狀態。

## D. 手機驗收（iOS 或 Android）
- [ ] D1 主要操作按鈕可正常觸控，無誤觸。
- [ ] D2 錯誤訊息不被虛擬鍵盤長時間遮擋。
- [ ] D3 欄位錯誤與摘要可讀性良好。
- [ ] D4 修正欄位後按鈕狀態即時更新。
